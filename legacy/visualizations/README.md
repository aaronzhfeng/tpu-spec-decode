# Visualizations — DFlash TPU Speculative Decoding

Two visualization suites + an interactive demo.

## Directory Structure

```
visualizations/
  README.md                          # this file
  DESIGN.md                          # design specs for research figures
  scripts/
    generate_benchmarks.py           # 13 benchmark result figures (V5P + V4)
    generate_research.py             # 7 research contribution figures
    capture_replay.py                # post-process benchmark JSONs to replay format
    run_all_replay.sh                # run all 9 benchmarks + capture replay data
  output/
    benchmarks/                      # benchmark result PNGs (figs 1-13)
    research/                        # research contribution PNGs (figs 1-7)
    replay/                          # live inference replay data
      raw_{dataset}.json             # full benchmark output with text
      replay_{dataset}.json          # replay-ready format with timestamps
  interactive/
    token_replay.html                # real-time AR vs DFlash inference demo
```

## Quick Start

```bash
pip install matplotlib seaborn numpy

# Generate all benchmark figures
python visualizations/scripts/generate_benchmarks.py

# Generate all research figures
python visualizations/scripts/generate_research.py

# Generate a single research figure
python visualizations/scripts/generate_research.py --figure k-flat

# Generate as PDF
python visualizations/scripts/generate_research.py --format png pdf
```

## Replay Capture (Live Inference Data)

Captures real inference runs with full text and per-step acceptance patterns for the interactive demo. Requires TPU + Docker.

```bash
# Run all 9 datasets (inside Docker, ~20 min total)
bash visualizations/scripts/run_all_replay.sh

# Or run a single dataset
# (inside Docker)
python benchmarks/standalone_dflash.py \
    --target-model Qwen/Qwen3-4B \
    --draft-model z-lab/Qwen3-4B-DFlash-b16 \
    --dataset gsm8k --max-samples 3 \
    --output-json visualizations/output/replay/raw_gsm8k.json

# Post-process raw JSONs to replay format (no Docker needed)
python visualizations/scripts/capture_replay.py --all --skip-benchmark
```

**Raw JSON** (`raw_{dataset}.json`) contains per-sample:
- `baseline_text`, `dflash_text`, `prompt_text` — full decoded output
- `acceptance_lengths` — per-step burst pattern (e.g. `[4, 2, 16, 3, ...]`)
- `baseline_tpot_ms`, `dflash_tpot_ms` — timing in milliseconds

**Replay JSON** (`replay_{dataset}.json`) adds:
- `baseline.token_timestamps_ms` — reconstructed per-token timestamps
- `dflash.step_timestamps` — per-step timestamps with tokens accepted per burst

All timestamps in **milliseconds** (ms), consistent with benchmark output.

Open `interactive/token_replay.html` in a browser. Currently uses synthetic sample data; will be updated to load replay JSONs from `output/replay/`.

## Data Sources

All data lives under `results/` in the project root:
- `results/v5p/` — TPU V5P standalone benchmarks (9 datasets), CSV summaries
- `results/v4/` — TPU V4 benchmarks, profiling, GPU comparisons
- `results/verify_context_scaling.json` — K-flat verification data (K=16→1024)
- `visualizations/output/replay/` — live inference replay data (9 datasets, 3 samples each)
